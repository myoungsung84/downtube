# Release Note

Downtube 저장소 전용 릴리즈 노트 작성 절차다.

## 사용 시점

- 사용자가 `rn`, `릴리즈노트`, `릴리즈 노트`, `release note`, `Windows 릴리즈 노트` 작성을 요청했을 때 사용한다.
- 기본 기준은 Windows 릴리즈 노트다.

## 작업 절차

1. 먼저 `docs/release/release-note-template-win.md` 내용을 직접 확인한다.
2. `package.json`을 열어 `version` 값을 직접 확인한다.
3. 템플릿 구조를 그대로 유지한 채 아래 4개만 채운다.
- `version`
- `added`
- `improved`
- `fixed`
4. 템플릿의 나머지 문구는 바꾸지 않는다.
5. 템플릿의 섹션 순서도 바꾸지 않는다.
6. 한국어 안내가 먼저, 영어 안내가 나중에 오는 순서를 유지한다.
7. 항목은 사용자 관점 문장으로 작성한다.
8. 내부 구현 디테일이나 파일명 나열 위주 설명은 넣지 않는다.

## 출력 규칙

- 최종 결과는 반드시 하나의 코드블록으로만 출력한다.
- 코드블록 밖 설명은 최소화한다.
- 가능하면 최종 결과만 바로 제공하고, 중간 해설은 생략한다.
- 템플릿 해설, 작업 과정 요약, 불필요한 배경 설명은 붙이지 않는다.
