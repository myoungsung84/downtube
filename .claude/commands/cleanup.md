# cleanup

머지 완료된 워킹트리와 로컬/리모트 브랜치를 정리한다. 아직 머지되지 않은 브랜치는 유지한다. 작업이 끝나면 정리 결과와 남아 있는 워킹트리/브랜치 상태를 요약해서 출력한다.

## 규칙

- 기준 브랜치는 현재 체크아웃된 브랜치가 아니라 기본 브랜치(`main` 우선, 없으면 `master`)로 본다.
- 기본 브랜치에 머지된 로컬 브랜치만 삭제한다.
- 현재 체크아웃 중인 브랜치와 기본 브랜치는 삭제하지 않는다.
- 연결된 워킹트리가 있는 브랜치는 먼저 워킹트리를 제거한 뒤 브랜치를 삭제한다.
- 기본 브랜치에 머지되지 않은 브랜치와 그 워킹트리는 보존한다.
- 삭제 전후로 사용자 변경사항이 있는지 확인하고, 변경사항이 있으면 삭제하지 않고 이유를 요약에 포함한다.
- 로컬 브랜치 삭제 후 대응하는 remote 브랜치(`origin/<branch>`)도 존재하면 함께 삭제한다.
- remote 삭제는 `git push origin --delete <branch>`로 수행한다.

## 실행 절차

1. 기본 브랜치를 확인한다. `main`이 있으면 `main`, 없으면 `master`를 사용한다.
2. `git worktree list --porcelain`으로 워킹트리와 연결 브랜치를 수집한다.
3. `git branch --format='%(refname:short)'`로 로컬 브랜치 목록을 수집한다.
4. 각 브랜치에 대해 `git merge-base --is-ancestor <branch> <base-branch>`로 머지 여부를 판단한다.
5. 머지된 브랜치 중 현재 브랜치/기본 브랜치를 제외하고 정리 대상을 고른다.
6. 정리 대상 브랜치에 연결된 워킹트리가 있으면, 워킹트리 경로에 미커밋 변경이 없는 경우에만 `git worktree remove <path>`로 제거한다.
7. 워킹트리 제거가 끝난 브랜치는 `git branch -d <branch>`로 삭제한다.
8. `git ls-remote --heads origin <branch>`로 remote 존재 여부 확인 후 있으면 `git push origin --delete <branch>`로 삭제한다.
9. 머지되지 않은 브랜치, 변경사항 때문에 건너뛴 워킹트리/브랜치는 그대로 둔다.
10. 마지막에 아래 형식으로 요약한다.

## 출력 형식

```text
정리 완료
- 기준 브랜치: <base-branch>
- 삭제한 워킹트리: <count>
- 삭제한 로컬 브랜치: <count>
- 삭제한 remote 브랜치: <count>
- 보존한 미머지 브랜치: <branch 목록 또는 없음>
- 변경사항으로 건너뛴 항목: <항목 목록 또는 없음>
- 남아 있는 워킹트리: <worktree 목록 또는 없음>
- 남아 있는 로컬 브랜치: <branch 목록>
```
